Изначальный подход с парсингом HML b SVG структурой отображения оказался очень хрупким.
Давай полностью его поменяем и перейдём на API!

API примеры и описание я сложил в @makets/api/README-API.md

Давай переделаем в нашем решениии:
1. Вместо парсинга примера maket.as-is.html, используем АПИ запросы (видимо нужно 3)
2. Авторизацию подставлять не нужно, будет использоваться текучая кука пользователя.
3. Примеры приведены полные, большинство заголовков тоже можно поустить, кроме важных, таких как тип контента
4. Реализовываем класс HuEvaApi, для выполнения запросов:
   * Он должен достаточно конфигурироваться чтобы использовался неизменяемым в проде (юзерскрипт) и на нашем моке, соглдасно мапингну урлов, описанному в @makets/api/README-API.md
   * Вотчинг запущен (не надо запускать), можно смотреть и тестировать http://localhost:3000/
5. В `dev.html` полностью должно быть проэмулировано как это будет работать в юзерскрипте, на предоставленных примерах API.
